# НАЧНИ ОТСЮДА

Перенос дизайна прототипа (тёмная/светлая темы, шафран, Sora+Hanken, анимации)
в твой проект **magnifique** (Next.js + Tailwind + shadcn + Prisma).

## Хорошие новости
- **Ничего ставить не надо.** `framer-motion`, `lucide-react`, `shadcn` уже есть в твоём `package.json`.
  Все анимации заработают сразу.
- Логику **не трогаем**: Prisma, auth, API остаются как есть. Меняем только внешний вид.

## Что в папке `handoff/`
| Файл | Назначение | Куда |
|---|---|---|
| `theme-tokens.css` | светлая (твой бренд) + тёмная палитры | в `src/app/globals.css` |
| `theme.tsx` | переключение тем (localStorage + класс `.dark`) | → `src/lib/theme.tsx` |
| `ThemeToggle.tsx` | переключатель в настройки | → `src/components/settings/` |
| `fonts.ts` | Sora + Hanken | в `src/app/layout.tsx` |
| `motion.tsx` | анимации (magnetic, springs, кольцо, бар) | → `src/lib/motion.tsx` |
| `EventCard.example.tsx` | образец одного компонента | пример |
| `reference/` | **исходники прототипа** — эталон вида | смотреть, не копировать дословно |
| `DATA_MAP.md` | карта: поля прототипа → твоя Prisma-схема | свериться |

## Как применить — 3 шага

### Шаг 1. Скачай папку `handoff` и положи в корень проекта
Рядом с `src/`, `package.json`. Это нужно, чтобы Claude Code мог её прочитать.

### Шаг 2. Открой Claude Code в проекте и вставь это:

```
Прочитай handoff/НАЧНИ_ОТСЮДА.md, handoff/DATA_MAP.md и handoff/reference/.
Переноси дизайн прототипа в наш проект. Соблюдай CLAUDE.md.

Жёсткие правила:
- framer-motion, lucide-react, shadcn УЖЕ установлены — новые пакеты НЕ ставь.
- НЕ трогай prisma, auth, API, серверные загрузчики данных. Меняй только вид компонентов.
- Данные бери из наших реальных моделей (см. handoff/DATA_MAP.md), НЕ из mock прототипа.
- Работай на ветке redesign/dark-saffron, по одному экрану на коммит, после каждого: npm run lint && npm run build.
- "use client" на компонентах с анимацией/состоянием.

Порядок (жди моего ОК после каждого пункта):
1. Темы: вставь handoff/theme-tokens.css в src/app/globals.css (светлая в :root, тёмная в .dark).
   Создай src/lib/theme.tsx, оберни приложение в <ThemeProvider>, добавь suppressHydrationWarning
   на <html> и анти-FOUC скрипт в <head> (код в комментарии theme.tsx).
2. Шрифты: подключи handoff/fonts.ts (Sora+Hanken через next/font).
3. Настройки: добавь handoff/ThemeToggle.tsx на страницу настроек.
4. Экран «Мероприятия»: перепиши вид по эталону reference/screens-manager.jsx и
   handoff/EventCard.example.tsx, данные — из БД, поля по DATA_MAP.md.
5. Дальше по одному экрану: страница мероприятия, дашборд, сотрудники, заявки, табель.

Показывай diff каждого шага перед коммитом.
```

### Шаг 3. После каждого шага проверь
- `npm run build` — собирается
- тема переключается, при перезагрузке нет белой вспышки
- экран на реальных данных

## Если что-то сломалось
| Симптом | Фикс |
|---|---|
| `useTheme must be inside ThemeProvider` | оберни приложение в `<ThemeProvider>` в `layout.tsx` |
| Белая вспышка при загрузке тёмной | добавь анти-FOUC `<script>` в `<head>` (код в `theme.tsx`) |
| `Cannot use motion in Server Component` | добавь `"use client"` в начало файла |
| Сайдбар не перекрашивается при смене темы | `key={theme}` на обёртке (так сделано в прототипе) |
| Hydration warning | `suppressHydrationWarning` на `<html>` |

Всё. Начинай с пункта 1 — он безопасный (только цвета), логику не задевает.
