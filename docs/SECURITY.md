# Чеклист безопасности

Запускать **перед каждым PR** (новый эндпоинт или изменение авторизации) и **перед каждым деплоем**.

---

## Чеклист на каждый новый API-маршрут

- [ ] `const session = await auth()` — первая строка обработчика, до любой логики
- [ ] Проверка наличия сессии: `if (!session?.user) return 401`
- [ ] Проверка роли если нужна: `if (session.user.role !== "manager") return 403`
- [ ] Входящее тело валидируется через Zod перед использованием
- [ ] Путь-параметры (`params.id`) не используются в запросах БД без проверки владения/доступа
- [ ] Ответ не содержит `password_hash`, `passport_data_enc`, `push_subscription`
- [ ] Нет прямых SQL-запросов (только Prisma — защита от SQL-инъекций)

## Чеклист перед деплоем на продакшн

### Переменные окружения

- [ ] `AUTH_SECRET` — не дефолтный, минимум 32 символа случайных данных
- [ ] `PASSPORT_ENCRYPTION_KEY` — 64 hex-символа, уникальный для окружения
- [ ] `DATABASE_URL` и `DIRECT_URL` — указывают на продакшн-БД
- [ ] `NEXTAUTH_URL` — точный URL продакшн-домена (без слеша в конце)
- [ ] VAPID ключи сгенерированы один раз и сохранены в безопасном месте
- [ ] Никакой `.env.local` не попал в git (проверить `.gitignore`)

### База данных

- [ ] Бэкап настроен (Supabase: Point-in-Time Recovery или pg_dump по расписанию)
- [ ] `npm run db:migrate` применена на продакшн (не `db:push` — он без истории)
- [ ] Проверить что сид не запускается автоматически в CI/CD

### Приложение

- [ ] `npm run build` проходит без ошибок
- [ ] `npm run lint` — без ошибок
- [ ] Все env-переменные присутствуют в окружении деплоя
- [ ] Проверить `/api/auth/session` — возвращает только нужные поля (не `password_hash`)
- [ ] Страница `/employees` недоступна для waiter (проверить вручную)
- [ ] Паспортные данные не возвращаются в листинге сотрудников

### 152-ФЗ

- [ ] Сервер приложения — в России
- [ ] БД — в России (или Supabase регион eu-central, согласованный с юристом)
- [ ] Форма согласия на обработку ПД подписана сотрудниками до ввода паспортных данных

## Что НЕ делать никогда

- Не логировать `password`, `passport_data`, `push_subscription` в консоль/логи
- Не возвращать `password_hash` в API-ответах
- Не хранить токены/секреты в `localStorage` (используется только cookie через NextAuth)
- Не отключать проверку роли для ускорения разработки
- Не делать эндпоинты без авторизации (даже GET) без явного обоснования
